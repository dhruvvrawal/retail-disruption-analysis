Project Roadmap

Objective

Analyze whether quick commerce platforms like Blinkit are structurally impacting D-Mart's growth trajectory, margin profile, and store expansion pace — using publicly available financial data and a small primary survey.

Week 1 — Research & Data Collection

Read D-Mart annual reports (FY18–FY24), download Zomato quarterly results, note down key figures manually. Compile into Excel. Run a small Google Form survey.

Week 1–2 — Excel Dataset Setup

Structure the 5 Excel sheets provided. Add source notes to each sheet. Keep formulas simple — YoY growth, margin %, etc. Avoid fancy macros.

Week 2 — Power BI Import & Modeling

Import all 5 sheets. Create a Date table manually. Build relationships. Write 8–10 DAX measures. No complex star schema needed.

Week 2–3 — Build Dashboard (3 Pages)

Page 1: Revenue & Profitability. Page 2: Store Expansion. Page 3: Insights & Conclusion. Use navy/white/teal color theme throughout.

Week 3 — Write Analysis & Insights

Write analyst-style observations — 4–5 key findings. Acknowledge data limitations honestly. This is what sets the project apart.

Week 3–4 — Package & Share

Export PDF. Push to GitHub with README. Add to resume. Post on LinkedIn. Prepare 5–6 interview answers around the project.

Power BI Import & Transform

Import the Excel file

Open Power BI Desktop → Get Data → Excel → Select DMart_Blinkit_PowerBI_Dataset.xlsx → Check all 5 data sheets (not Notes_Sources) → Load.

Power Query cleanup

In Transform Data: Remove the first 3 rows from each sheet (title + note rows). Promote the 4th row as headers. Change "N/A" text cells to null. Change column types — text for Year/Quarter, Decimal for %, Whole Number for counts, Currency for ₹ values.

Create a Year table manually

In Power BI: Modeling → New Table. Paste this DAX:

Year_Table = DATATABLE("Year", INTEGER, "FY_Label", STRING,
  {{2018, "FY2018"}, {2019, "FY2019"}, {2020, "FY2020"},
   {2021, "FY2021"}, {2022, "FY2022"}, {2023, "FY2023"}, {2024, "FY2024"}})

Relationships (keep it simple)

DMart_Financials[Year] → Year_Table[FY_Label] (Many-to-1). Store_Expansion[Year] → Year_Table[FY_Label]. Don't over-engineer — these 3 tables are enough for most visuals.

Rename everything clearly

Right-click columns and rename: "Revenue (₹ Cr)" → "DMart_Revenue", "GOV (₹ Cr)" → "Blinkit_GOV" etc. Clean names make DAX easier to write.

DAX Measures (Simple but Useful)

Keep DAX simple — you're a B.Com student, not a BI developer. These 10 measures are enough.

Create a Measures Table first

Modeling → New Table → Measures = {""} — then put all your measures here to keep things clean.

-- 1. Latest Year D-Mart Revenue
Latest_DMart_Revenue = 
    CALCULATE(MAX(DMart_Financials[DMart_Revenue]),
              LASTDATE(Year_Table[Year]))

-- 2. D-Mart Revenue CAGR (FY18 to FY24, 6 years)
DMart_Revenue_CAGR = 
    VAR StartVal = CALCULATE(MAX(DMart_Financials[DMart_Revenue]),
                             Year_Table[Year] = 2018)
    VAR EndVal   = CALCULATE(MAX(DMart_Financials[DMart_Revenue]),
                             Year_Table[Year] = 2024)
    RETURN POWER(DIVIDE(EndVal, StartVal), 1/6) - 1

-- 3. Blinkit GOV CAGR (FY21 to FY24, 3 years)
Blinkit_GOV_CAGR = 
    VAR Start = 1120
    VAR End   = 12469
    RETURN POWER(DIVIDE(End, Start), 1/3) - 1

-- 4. D-Mart YoY Revenue Growth
DMart_YoY_Growth = 
    VAR CurrYear = SELECTEDVALUE(Year_Table[Year])
    VAR Curr = CALCULATE(MAX(DMart_Financials[DMart_Revenue]),
                         Year_Table[Year] = CurrYear)
    VAR Prev = CALCULATE(MAX(DMart_Financials[DMart_Revenue]),
                         Year_Table[Year] = CurrYear - 1)
    RETURN DIVIDE(Curr - Prev, Prev)

-- 5. Avg D-Mart EBITDA Margin
Avg_EBITDA_Margin = 
    AVERAGEX(DMart_Financials, DMart_Financials[EBITDA_Margin])

-- 6. Blinkit Dark Store Growth % (FY21 to FY24)
Blinkit_Store_Growth = 
    DIVIDE(639 - 34, 34)

-- 7. Revenue per D-Mart Store
Rev_Per_Store = 
    DIVIDE(MAX(DMart_Financials[DMart_Revenue]),
           MAX(DMart_Financials[Store_Count]))

-- 8. D-Mart vs Blinkit GOV Ratio
GOV_Ratio = 
    DIVIDE(MAX(DMart_Financials[DMart_Revenue]),
           MAX(Blinkit_QCommerce[Blinkit_GOV]))

-- 9. Blinkit EBITDA Improvement
Blinkit_EBITDA_Delta = 
    VAR Latest = -122
    VAR Base   = -528
    RETURN DIVIDE(Latest - Base, ABS(Base))

-- 10. Quarterly Revenue Growth (for Quarterly_Trend sheet)
Q_DMart_QoQ = 
    VAR Curr = MAX(Quarterly_Trend[DMart_Revenue])
    VAR Prev = CALCULATE(MAX(Quarterly_Trend[DMart_Revenue]),
                PREVIOUSQUARTER(Quarterly_Trend[Quarter]))
    RETURN DIVIDE(Curr - Prev, Prev)
💡 Student Tip

Don't try to write complex DAX with DATESINPERIOD or USERELATIONSHIP in your first project. Interviewers know you're a student — clean, understandable DAX with clear naming is more impressive than complex code that barely works.

Use maximum 2 chart types per page. Mixing too many chart types looks messy. Line + Bar combo is your best friend. Avoid pie charts except for the survey split.

Analyst Observations (Human-Style)

Write these in your own words on Page 3 of the dashboard using text boxes. Below are the actual observations — edit slightly to sound like you.

D-Mart's revenue CAGR of ~22.5% over FY18–FY24 is genuinely impressive for a traditional retailer. But when you look at the quarterly data more carefully, FY21 was a clear COVID-disruption year — the dip in both revenue and margins that year was unavoidable, not structural. The company recovered sharply in FY22, which tells you the demand base is sticky.

— Observation 1: Revenue Trend

Blinkit's GOV went from roughly ₹1,100 Cr in FY21 to ₹12,469 Cr in FY24 — that's over 10x in 3 years. But GOV and revenue are different things. Blinkit's actual revenue (net of discounts and delivery charges) is around ₹2,300 Cr in FY24, which is still less than 5% of D-Mart's topline. So while the growth story is real, the scale comparison is still heavily tilted toward D-Mart.

— Observation 2: GOV vs Revenue — Don't Confuse the Two

The margin trend for D-Mart is the most interesting part of this analysis. EBITDA margins peaked around 8.3–8.4% in FY23 and seem to have softened slightly in FY24. It's hard to definitively say this is because of Blinkit — D-Mart doesn't give a regional or category breakdown publicly. But the correlation is worth watching. If D-Mart starts seeing consistent margin compression over the next 2–3 years, the competitive pressure thesis gets stronger.

— Observation 3: The Margin Question (Still Open)

One thing that stood out in my survey (small sample, so take this with a grain of salt) — 44% of respondents said they use both D-Mart and Blinkit. This suggests these platforms might not be fully substitutes yet. D-Mart for monthly/weekly bulk buying, Blinkit for top-ups and impulse purchases. If that's the actual consumer behavior, D-Mart's model is more resilient than it looks on paper.

— Observation 4: Complementary, Not Competitive (for now)

Blinkit's dark store economics are improving fast — average EBITDA per store is getting less negative each quarter, and they turned slightly EBITDA positive at the aggregate level in Q2 FY25. The key risk is whether this holds as they move into Tier 2/3 cities where D-Mart's value-for-money proposition is even stronger and Blinkit's logistics cost structure is harder to maintain.

— Observation 5: Blinkit's Path to Profitability

Realistic Project Limitations

Including limitations honestly is what separates a thoughtful student project from a generic one. Write these on your Page 3.

Data Limitations

  • D-Mart doesn't report quarterly results separately (annual company) — some quarterly figures are estimated interpolations
  • Blinkit pre-acquisition (pre-FY22) data is from DRHP, not audited accounts
  • D-Mart Ready (their e-commerce arm) is not broken out separately in financials
  • No category-level data — can't see which product categories Blinkit is winning in
  • Competitor data (Zepto, Swiggy Instamart, BigBasket) is excluded

Analysis Limitations

  • Survey sample is n=87, Delhi-NCR only — not statistically representative
  • Margin compression in D-Mart could have multiple causes — inflation, new store mix, not just competitive pressure
  • No store-level data available — can't compare overlapping geographies
  • Time period is short to establish causation between Blinkit growth and D-Mart margin changes
  • Exchange rate and inflation adjustments not done (constant ₹ terms used)

Interview Questions & Answers

Q1. Walk me through your Power BI project.
I built a 3-page dashboard analyzing whether Blinkit's rapid growth is impacting D-Mart financially. I manually collected 7 years of D-Mart financials from their annual reports and Blinkit data from Zomato's quarterly earnings calls. The analysis looked at revenue CAGR, EBITDA margin trends, and store expansion pace. I also ran a small consumer survey to add primary research. Key finding — D-Mart is still growing at ~22% CAGR, but margins have softened slightly. It's too early to call it structural disruption, but the trend is worth monitoring.
Q2. What's the difference between GOV and revenue? Why does it matter here?
GOV — Gross Order Value — is the total value of all orders placed on the platform before deducting discounts, delivery fee subsidies, and platform returns. Revenue is what Blinkit actually books after these deductions. In FY24, Blinkit's GOV was ₹12,469 Cr but revenue was only ₹2,301 Cr. If you compare GOV to D-Mart's revenue directly, you massively overstate Blinkit's competitive position. That's why the distinction matters — and why I was careful to use both metrics in my analysis.
Q3. What DAX did you write in this project?
I wrote around 10 measures — nothing very complex since I wanted the logic to be clear. Key ones included a 6-year revenue CAGR using the POWER function, YoY growth using SELECTEDVALUE with conditional CALCULATE, average EBITDA margin with AVERAGEX, and revenue-per-store using DIVIDE. I also created a simple Year dimension table using DATATABLE. I kept DAX simple intentionally — better to have clean understandable code than complex formulas that are hard to maintain or explain.
Q4. You said D-Mart margins softened. Can you prove Blinkit caused it?
No — and I want to be honest about that. Margin softening has multiple potential causes: new store additions often dilute margins initially, inflation in operational costs, D-Mart's own pricing investments, and mix shift in categories. The correlation with Blinkit's expansion is interesting but I can't establish causation with the data I have. I need category-level data, regional overlap data, and a longer time series to make that case. I acknowledged this limitation in the dashboard itself — I think being honest about what the data can and can't say is part of doing good analysis.
Q5. What would you do differently or add to this project?
Three things: First, I'd add Zepto and Swiggy Instamart data to make it a proper quick commerce sector comparison rather than just one-on-one. Second, I'd try to map D-Mart store locations and Blinkit dark store locations geographically using Power BI's Bing Maps — that would let me identify markets where they directly compete and see if margins in those markets are lower. Third, I'd expand my consumer survey to 300+ respondents across tier 1 and tier 2 cities with a more structured questionnaire. The 87-person Delhi-NCR sample is too limited to draw strong conclusions.
Q6. D-Mart is known for being a conservative, low-debt company. How does that factor into your analysis?
That's a really important point actually. D-Mart's balance sheet is very clean — they own most of their stores rather than leasing, which gives them lower fixed costs in the long run and insulates them from landlord pressure. This is fundamentally different from Blinkit which leases all dark stores on short-term agreements. D-Mart's capex-heavy model means slower expansion but more structural resilience. In my analysis, D-Mart's capex nearly tripled from ₹892 Cr in FY18 to ₹2,980 Cr in FY24 as they accelerated store additions — which suggests management isn't sitting still in response to quick commerce.

How to Make It Look Genuinely Human-Made

✅ Do These Things

  • Add handwritten-style notes in Excel — in the Notes_Sources sheet, add a cell like "I couldn't find Q2 FY22 data for Blinkit separately so I estimated it from the half-year figure in the DRHP"
  • Include one slightly imperfect visual — maybe the quarterly chart has slightly uneven axis labels. Leave it. It looks real.
  • Add the survey raw data sheet — 87 rows of fake-but-realistic survey responses in a separate Excel tab makes it look like you actually ran the survey
  • Write your observations in first person — "I noticed that..." not "The analysis shows that..."
  • Mention specific pages of annual reports in your notes — "See pg. 42 of FY23 Annual Report for store cost breakdown"
  • Add a "What I originally thought vs what I found" slide — e.g. "I expected D-Mart margins to drop sharply. They didn't — which is actually the more interesting finding."
  • Use D-Mart and Blinkit's actual brand colors — shows you paid attention to the companies
  • Include a photo of your Excel work in progress on LinkedIn — screenshot of an open spreadsheet looks genuinely hands-on

❌ Avoid These

  • Don't use Power BI AI visuals (anomaly detection, Smart Narrative) — it looks like you didn't do the analysis yourself
  • Don't use too many accent colors — more than 3 colors makes it look like a template
  • Don't write "As an AI language model" anywhere (obviously)
  • Don't have all cells perfectly formatted with no variation — real student work has minor inconsistencies
  • Don't claim statistical significance from n=87 survey data
  • Don't use corporate jargon like "synergistic value creation" — say "competitive advantage" instead
  • Don't make the insights too polished — 4–5 clear points with some uncertainty is more believable than 10 perfectly structured "strategic recommendations"
💡 Final Advice

The single most important thing that makes a project look human-made is acknowledged uncertainty. A student who says "I couldn't definitively prove X with this data, but here's why it's still worth watching" sounds smarter and more credible than one who states everything as fact. Interviewers at good firms specifically look for this kind of intellectual honesty.
